Which AI tool should Greg use?

An interactive field guide to Google AI Studio, Codex, Claude Cowork, and Lovable. The core distinction: Google AI Studio is the Gemini lab, Codex is the engineering agent, Claude Cowork is the operating system, and Lovable is the fastest web-product builder.

Gemini Lab

Google AI Studio

Use when the question is: Can Gemini do this?

  • Prompt and model experiments
  • Gemini API starter code
  • Multimodal tests
  • Google-native prototypes
Engineering Agent

Codex

Use when the question is: Can this code be made correct?

  • Repo inspection and edits
  • Testing and verification
  • Code review and debugging
  • AGENTS.md, skills, plugins, MCP
Operating System

Claude Cowork

Use when the question is: What are we doing and why?

  • Board personas

Product Builder

Lovable

Short White Paper

Recommendation

Do not add Google AI Studio as a fifth general-purpose workspace. Add it as a Gemini lab. It solves direct, first-party experimentation with Google's Gemini models and Google-native app integrations. What you remove is the habit of using Lovable or Claude Cowork to guess what Gemini can do.

Default workflow

Claude Cowork decides priority. Lovable drafts the web product. Codex audits and hardens code. Google AI Studio enters only when Gemini-specific capability is the point.

Complexity test

Before adding a tool to the regular stack, ask: What does this solve that the current stack cannot? What are we removing or narrowing to make room for it?
